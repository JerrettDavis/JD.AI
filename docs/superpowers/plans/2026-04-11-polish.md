# Polish & Cleanup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add consistent loading skeletons and error boundaries to all MVP pages, then remove legacy UI surfaces (OpenClaw stat card, bridge table, OpenClaw settings tab, non-MVP nav links, and channel-type filter in Chat) that fall outside the JD.AI MVP spec.
**Architecture:** Two shared Razor components (`LoadingSkeleton` and `GatewayErrorBoundary`) placed in `Components/Shared/` and consumed by each of the 10 MVP pages; legacy removal is surgical edits to `Home.razor`, `Settings.razor`, `NavMenu.razor`, and `Chat.razor`, each driven by a failing bunit test first. Playwright features are updated before implementation to ensure TDD coverage.
**Tech Stack:** Blazor WASM .NET 10, MudBlazor v8.5.1, xunit/bunit, Playwright/Reqnroll

---

## Issue Context

**#479** — Loading skeletons and error boundaries must cover all MVP pages. Currently only Home (stat cards), Agents (rows), Logs (rows), Plugins (rows), Providers (cards), and Settings (panel) have skeletons. Chat lacks an agent-load skeleton; Skills page has no skeleton; Settings sub-tabs have no per-tab error states. No page uses a reusable error boundary component — errors are either silent (Agents) or inline alerts without retry.

**#480** — Legacy UI surfaces to remove:
- `Home.razor`: 4th stat card is `data-name="openclaw"` (OpenClaw bridge status) — replace with Gateway connection status card aligned to MVP spec.
- `Home.razor`: `[data-testid="openclaw-bridge"]` table section — remove entirely.
- `Settings.razor`: "OpenClaw" `MudTabPanel` and its `SettingsOpenClawTab` component — remove the tab; component file can remain but stop rendering it.
- `NavMenu.razor`: Non-MVP nav links — `/channels`, `/sessions`, `/providers`, `/routing`, `/monitor`, `/analytics`, `/knowledge`, `/apikeys` are all deferred/out-of-scope; remove them. Keep: `/` (Overview), `/chat`, `/agents`, `/plugins` (Skills), `/logs`, `/settings`.
- `Chat.razor`: Channel-type filter `MudSelect` (`data-testid="channel-selector"`) — remove. Also remove `"signal"` and `"openclaw"` entries from `ChannelIcons` dict; remove the `_selectedChannel` field and `_filteredSessions` computed prop.

## Step 4 — Implement GatewayErrorBoundary component

- [ ] Create `src/JD.AI.Dashboard.Wasm/Components/Shared/GatewayErrorBoundary.razor`:

  ```razor
  @* Reusable error boundary for gateway connection failures.
     Shows a MudAlert with retry button when HasError is true.
     Renders ChildContent passthrough when HasError is false. *@

  @if (HasError)
  {
      <MudAlert Severity="Severity.Error" Variant="Variant.Outlined"
                data-testid="gateway-error-alert" Class="mt-4">
          <MudStack Row="true" AlignItems="AlignItems.Center" Justify="Justify.SpaceBetween">
              <MudText data-testid="gateway-error-message">@Message</MudText>
              <MudButton Variant="Variant.Text" Color="Color.Error" Size="Size.Small"
                         StartIcon="@Icons.Material.Filled.Refresh"
                         OnClick="OnRetry"
                         data-testid="gateway-retry-button">
                  Retry
              </MudButton>
          </MudStack>
      </MudAlert>
  }
  else
  {
      @ChildContent
  }

  @code {
      [Parameter] public bool HasError { get; set; }
      [Parameter] public string Message { get; set; } = "Gateway unreachable. Is the JD.AI Gateway running?";
      [Parameter] public EventCallback OnRetry { get; set; }
      [Parameter] public RenderFragment? ChildContent { get; set; }
  }
  ```
