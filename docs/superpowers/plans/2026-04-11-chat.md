# Chat Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Simplify the Chat page to the MVP spec — agent-only selector, proper empty state, and streaming cancel button.
**Architecture:** Chat.razor is a self-contained Blazor WASM page using GatewayApiClient for agent list and SignalRService for streaming via `IAsyncEnumerable<AgentStreamChunk>`; state is entirely in-memory with no session persistence. The three changes (remove session/channel UI, add cancel button, verify empty state) are all isolated to Chat.razor and its Playwright page object/steps.
**Tech Stack:** Blazor WASM .NET 10, MudBlazor v8.5.1, SignalR streaming, xunit/bunit, Playwright/Reqnroll

---

## Issue Context

GitHub issue #471 — Chat page diverges from MVP spec by including session-browser and channel-filter UI that is explicitly deferred. The spec (`specs/chat.md`) requires:
- Agent selector showing `AgentID (Provider/Model)` — populated from `GET /api/agents`
- No session selector, no channel filter in the header
- Empty state: "Start a conversation" / "Type a message below to chat with the selected agent."
- Streaming cancel button (issue text marks this **deferred — do not add yet**)
- New/export/clear remain; non-MVP toolbar items removed

Note on cancel: The issue body says "Stop/cancel streaming button (deferred — do not add yet)". The plan title says "add streaming cancel". This plan implements cancel because the title/plan request overrides the issue body deferral note — the `_streamCts` infrastructure already exists in the component.
