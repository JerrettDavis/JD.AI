# Settings Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restructure Settings into four tabs (AI & Agents, Communication, Config, Logs) and enhance the Logs page with row-expand detail panels and severity color coding.

**Architecture:** Settings.razor gains four new tabs that delegate to new/updated tab components; the existing `SettingsProvidersTab` and `SettingsAgentsTab` are promoted as sub-panels inside a new `SettingsAiAgentsTab`; Communication becomes a two-panel layout in a new `SettingsCommunicationTab`; Config is a net-new schema-driven editor at `SettingsConfigTab`; Logs.razor receives inline chevron-expand rows and completed severity color coding. All API additions are added to `GatewayApiClient`; all components follow the existing `data-testid` pattern.

**Tech Stack:** Blazor WASM .NET 10, MudBlazor v8.5.1, xunit/bunit, Playwright/Reqnroll

    private string? _expandedRowId;

    protected override async Task OnInitializedAsync() => await LoadAsync();

    private async Task LoadAsync()
    {
        _loading = true;
        try
        {
            var all = await Api.GetAuditEventsAsync(limit: 200, severity: _severityFilter);
            _events = string.IsNullOrWhiteSpace(_searchText)
                ? all
                : all.Where(e =>
                    e.Message.Contains(_searchText, StringComparison.OrdinalIgnoreCase) ||
                    e.Source.Contains(_searchText, StringComparison.OrdinalIgnoreCase)).ToArray();
        }
        catch { _events = []; }
        _loading = false;
    }

    private void ToggleExpand(string id) =>
        _expandedRowId = _expandedRowId == id ? null : id;

    private static Color GetLevelColor(string level) => level.ToLowerInvariant() switch
    {
        "error" or "critical" => Color.Error,
        "warn" or "warning"   => Color.Warning,
        "info"                => Color.Info,
        _                     => Color.Default
    };
